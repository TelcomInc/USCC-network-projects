Deployment

Cloudflare migration checklist

The static page can move to Cloudflare Pages now. The full app should use Cloudflare Access, Workers, D1, and R2.
Pages

Connect the GitHub repo, deploy from main, leave build command blank, and publish the repository root.

Access

Protect the app before client data is uploaded. Do not rely on JavaScript passwords in a public static site.

Data

Use D1 for project/device records and R2 for plans, workbooks, test files, warranties, and generated packets.

Domain Automation
Recommended first version
For company-owned domains, the app can create client subdomains with Cloudflare DNS API records. For buying brand-new client domains, keep purchase and registrant contact confirmation in the Cloudflare dashboard until the business flow is locked down.
